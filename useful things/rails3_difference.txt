Changes between Rails-2 & Rails-3

1) In Rails - 2, Instance variable are allowed as method argument & in looping structure. But in Rails-3, Instance variable are not allowed as method argument & in looping structure.

2) Callbacks are changed.
   Rails - 2 :- 
      def after_destroy
        some code
      end
   Rails - 3 :-
      after_destroy do
        some code
      end

3) New way of writing callbacks :-
   after_destroy : some_method_name()
   def some_method_name()
     some code
   end

4) Notifier syntax changed :-
   RAILS-2 :- DspMailer.deliver_notification(dsp::DSP_EMAIL[RAILS_ENV.to_sym], message)
   RAILS-3 :- DspMailer.notification(dsp::DSP_EMAIL[Rails.env.to_sym], message).deliver

   RAILS-2 :- 
              mail = Notifier.create_notify_user(album.artist.name, 'gagan.shah@cipher-tech.com', album.title, dsp_name, album.upc)
              Notifier.deliver(mail)
   RAILS-3 :- 
              mail = Notifier.notify_user(album.artist.name, 'benp@dashgo.com', album.title, dsp_name, album.upc)
              mail.deliver

5) Changes in Ajax Calling

6) Previously if we write any HTML tags in string, it will directly print HTML tags. But in Rails-3, we have to use "<b>gagan</b>".html_safe method.

7) RAILS_ROOT is changed to Rails.root.to_s & RAILS_ENV is changed to Rails.env

8) To count hash length, previously we are using 'length' method, now we have to use 'count' mthod.

10) Lambda :-
  Ruby 1.8.7 :- c = lambda {|a,b| a*b }
  Ruby 1.9.2 :- c = ->(a,b){ a*b }

11)Implicit Call :-
  Ruby 1.8.7 :- 
    c = lambda {|a,b| a*b }
    c.call(3,4)

  Ruby 1.9.2 :- 
    c = lambda {|a,b| a*b }
    c.(3,4)
